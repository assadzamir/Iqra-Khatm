  Implement 7 mutation hooks with exact signatures:

  **`useCreateKhatm(): UseMutationResult<KhatmGroup, Error, CreateKhatmInput>`**:
  1. `supabase.rpc('generate_invite_code')` ŌåÆ `invite_code`; if error after 10 retries (Postgres raises exception): reject with "Could not create group. Please try again."
  2. `supabase.from('khatm_groups').insert({...input fields, invite_code, admin_user_id: auth.uid(), status: 'ACTIVE', khatm_cycle: 1}).select().single()` ŌåÆ `group`
  3. Insert creator row: `khatm_participants` with `group_id: group.id, user_id: auth.uid(), name: currentUserName, role: 'ADMIN', status: 'JOINED', joined_at: new Date().toISOString()`
  4. Insert `reminder_windows.length` rows into `khatm_reminder_schedules` (one per `days_before` value in `input.reminder_windows`)
  5. `onSuccess`: `queryClient.invalidateQueries({ queryKey: khatmKeys.groups() })`

  **`useAssignJuz(): UseMutationResult<KhatmJuzAssignment, Error, AssignJuzInput>`** where `AssignJuzInput = { group_id: string; participant_id: string; juz_number: number; assigned_by: string; notify: boolean }`:
  - Insert into `khatm_juz_assignments`; insert audit log `action_type: 'JUZ_ASSIGNED'`
  - If `input.notify`: fire push notification (fire-and-forget)
  - On 409/unique constraint: map to "This Juz already has the maximum number of assignees."
  - `onSuccess`: `queryClient.invalidateQueries({ queryKey: khatmKeys.screen(input.group_id) })`

  **`useUpdateProgress(): UseMutationResult<void, Error, UpdateProgressInput>`** where `UpdateProgressInput = { assignment_id: string; group_id: string; participant_id: string; progress_percent: number; previous_percent: number; source: ProgressSource; note?: string }`:
  - Insert `khatm_progress_updates` row
  - Update `khatm_juz_assignments SET progress_percent = input.progress_percent WHERE id = input.assignment_id`
  - On error: reject with "Failed to save progress. Please try again."
  - `onSuccess`: `queryClient.invalidateQueries({ queryKey: khatmKeys.screen(input.group_id) })`

  **`useAssignRole(): UseMutationResult<void, Error, AssignRoleInput>`** where `AssignRoleInput = { participant_id: string; group_id: string; new_role: ParticipantRole; keep_records?: boolean; admin_participant_id: string }`:
  - UPDATE `khatm_participants SET role = input.new_role WHERE id = input.participant_id`
  - If `input.keep_records === false` (demoting CO_ADMIN): UPDATE `khatm_progress_updates SET participant_id = input.admin_participant_id WHERE participant_id = input.participant_id AND source = 'ADMIN_OVERRIDE'`
  - Insert audit log: `action_type: input.new_role === 'CO_ADMIN' ? 'ROLE_ASSIGNED' : 'ROLE_REVOKED'`
  - On PARTICIPANT status not JOINED: reject with "Cannot promote inactive members."
  - `onSuccess`: `queryClient.invalidateQueries({ queryKey: khatmKeys.screen(input.group_id) })`

  **`useJoinKhatm(): UseMutationResult<{ groupId: string }, Error, { invite_code: string; name: string }>`**:
  - SELECT `khatm_groups WHERE invite_code = input.invite_code`; if no row: reject with "Invalid code. Please check and try again."
  - Check existing participant: if row exists with same `group_id + user_id`: resolve with `{ groupId }` (not an error ŌĆö navigate to group)
  - Insert `khatm_participants` row: `role: 'PARTICIPANT', status: 'JOINED', joined_at: now(), name: input.name, user_id: auth.uid()`
  - Insert audit log `action_type: 'MEMBER_JOINED'`
  - `onSuccess`: `queryClient.invalidateQueries({ queryKey: khatmKeys.groups() })`

  **`useStartNewCycle(): UseMutationResult<KhatmGroup, Error, { source_group_id: string }>`**:
  - Fetch source group; get `khatm_cycle`
  - Call `generate_invite_code()` RPC for new code
  - Insert new `khatm_groups` row: all same settings, `khatm_cycle: oldGroup.khatm_cycle + 1`, `status: 'ACTIVE'`, new `invite_code`, `completed_at: null`
  - Copy all JOINED participants: INSERT batch into `khatm_participants` with `joined_at: now()`
  - On error: reject with "Failed to start new cycle. Please try again."
  - `onSuccess`: `queryClient.invalidateQueries({ queryKey: khatmKeys.groups() })`

  **`useUpdateGroupSettings(): UseMutationResult<void, Error, UpdateGroupSettingsInput>`** where `UpdateGroupSettingsInput = { group_id: string; actor_participant_id: string; updates?: Partial<Pick<KhatmGroup, 'assignment_mode' | 'max_per_juz' | 'allow_juz_switch'>>; reminder_windows?: number[]; regenerate_invite?: boolean }`:
  - If `input.updates`: UPDATE `khatm_groups SET ...` for provided fields
  - If `input.reminder_windows`: UPDATE all existing `khatm_reminder_schedules SET is_active = false WHERE group_id = input.group_id`; then INSERT new rows for each `days_before` value
  - If `input.regenerate_invite`: RPC `generate_invite_code()`, UPDATE `khatm_groups SET invite_code = newCode`
  - Insert audit log entries for each changed field
  - `onSuccess`: `queryClient.invalidateQueries({ queryKey: khatmKeys.screen(input.group_id) })`

- **Acceptance Criteria**:
  1. WHEN `useCreateKhatm` is called with `reminder_windows: [5, 2, 1]`, exactly 1 `khatm_groups` row, 1 `khatm_participants` row (ADMIN), and 3 `khatm_reminder_schedules` rows are inserted
  2. WHEN `useJoinKhatm` is called with an `invite_code` that matches no row in `khatm_groups`, the mutation rejects with the message "Invalid code. Please check and try again."
  3. WHEN `useAssignJuz` fails due to UNIQUE constraint (Postgres 23505 error code), the error message "This Juz already has the maximum number of assignees." is returned to the caller
